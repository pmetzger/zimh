# AT&T 3B2

This documentation is under development and currently incomplete.

The AT&T 3B2 simulator family currently builds two executables:

| Target | System | CPU |
| --- | --- | --- |
| `3b2-400` | AT&T 3B2/400 | WE32100 |
| `3b2-700` | AT&T 3B2/700 | WE32200 |

The simulator code was originally written by Seth Morabito. Additional
3B2 emulator information and software archives are available from:

- <https://loomcom.com/3b2/>
- <https://loomcom.com/3b2/emulator/>

## Current ROM Handling

The 3B2 targets do not yet compile ROM images into the executable. Before
booting a ROM-based environment, load the correct ROM image explicitly:

```text
sim> load -r 3b2_400.bin
sim> boot
```

The 3B2/400 maintenance smoke test uses
`simulators/3B2/tests/artifacts/3b2_400.bin`. The 3B2/700 ROM is also
committed as `simulators/3B2/tests/artifacts/3b2_700.bin`, but it is not
yet wired into automated 3B2/700 boot tests.

This is expected to change later. The planned direction is to handle 3B2
ROMs like the VAX-family simulators, where ROM images are converted into
generated C headers and compiled into the simulator.

## Memory

The `CPU` device controls the configured memory size.

`3b2-400` supports:

```text
sim> set cpu 512K
sim> set cpu 1M
sim> set cpu 2M
sim> set cpu 3M
sim> set cpu 4M
```

`3b2-700` supports:

```text
sim> set cpu 4M
sim> set cpu 8M
sim> set cpu 16M
sim> set cpu 32M
sim> set cpu 64M
```

The default is 4 MiB for `3b2-400` and 16 MiB for `3b2-700`.

## Devices

Both targets include the common CPU, MMU, MAU, interval timer, DMA,
integrated DUART, integrated floppy controller, NVRAM, time-of-day clock,
SCSI host adapter, Ethernet, and PORTS serial expansion support. The exact
system-board details differ between the 400 and 700 models.

`3b2-400` additionally includes an integral MFM fixed disk controller
(`IDISK`) and cartridge tape controller (`CTC`). `3b2-700` uses the Rev 3
system-board path and does not include the `IDISK` or `CTC` devices in its
current device list.

Important device names:

| Device | Meaning |
| --- | --- |
| `CPU` | WE32100 or WE32200 CPU and memory configuration |
| `MMU` | WE32101 or WE32201 memory-management unit |
| `MAU` | WE32106 math accelerator |
| `TMR` | Programmable interval timer |
| `NVRAM` | Battery-backed configuration and diagnostic RAM |
| `TOD` | Time-of-day clock |
| `IFLOPPY` | Integrated floppy controller |
| `IDISK` | 3B2/400 integral MFM fixed disk controller |
| `SCSI` | CM195W SCSI host adapter |
| `NI` | CM195A Ethernet controller |
| `PORTS` | CM195B 4-port serial multiplexer |
| `CTC` | 3B2/400 cartridge tape controller |

Use `HELP <device>` and `HELP <device> ATTACH` inside the simulator for
the detailed command syntax implemented by each device.

## Persistent State

The `NVRAM` device stores system configuration and diagnostic data. Attach
it to a file if the guest should preserve that state between simulator
runs:

```text
sim> attach nvram 3b2.nvram
```

The `TOD` device may also be attached to a file to preserve simulated clock
state:

```text
sim> attach tod 3b2.tod
```

## Floppy And Maintenance Utilities

The integrated floppy controller is named `IFLOPPY`. The integration
tests for the build include the AT&T 3B2 Maintenance Utilities
4.0 diskette image, which can be attached as follows:

```text
sim> attach -r ifloppy 3B2_Maintenance_Utilities_4-0.img
```

The image is a 720 KiB sector-by-sector diskette image. The Loomcom archive
describes these media as double-sided, 80-track, 96 tpi DS/DD, 250 kbps
MFM diskettes with 9 sectors per track and 3:1 interleave.

## Fixed Disk And SCSI Media

The 3B2/400 integral disk controller is `IDISK`. Supported drive type
commands include:

```text
sim> set idisk0 hd30
sim> set idisk0 hd72
sim> set idisk0 hd72c
sim> set idisk0 hd135
sim> set idisk0 hd161
```

`HD161` requires large-disk support:

```text
sim> set idisk large
```

Attach disk images with the normal simulator `ATTACH` command. If the file
does not exist, the simulator creates it unless `-e` is specified.

```text
sim> set idisk0 hd135
sim> attach -n idisk0 empty-hd135.img
```

The SCSI host adapter supports disk and tape device types including
`SD155`, `SD300`, `SD327`, `SD630`, and `ST120`.

## Ethernet

The Ethernet controller is `NI`. It is disabled at startup. Enable it, set
a MAC address if needed, and attach it to a host network backend:

```text
sim> set ni enable
sim> show eth
sim> attach ni eth0
```

The default MAC address is self-assigned in the AT&T range. Guest software
may set the address itself. Host networking support depends on the normal
zimh/SIMH networking backends available on the build platform.

## Example: Boot The 3B2/400 Maintenance Diskette

This example follows the same broad shape as the build's automated
smoke test, but uses local file names:

```text
sim> set cpu 4M
sim> load -r 3b2_400.bin
sim> attach nvram 3b2_400.nvram
sim> set idisk0 hd135
sim> attach -n idisk0 empty-hd135.img
sim> attach -r ifloppy 3B2_Maintenance_Utilities_4-0.img
sim> boot
```

The ROM should reach the maintenance-program prompt. From there, booting
`dgmon` enters the diagnostic monitor. Do not run disk or floppy formatting
utilities unless you are intentionally working with disposable media.

## Tests And Artifacts

3B2 test artifacts live under `simulators/3B2/tests/artifacts/`.

## Current Limitations

- ROM images must currently be loaded explicitly with `LOAD -r`.
- The 3B2/700 boot path has a committed ROM artifact, but does not yet
  have a real-software integration boot test.
- The 3B2/400 Maintenance Utilities test is intentionally only a smoke
  test; it does not run destructive or long-running diagnostics.
- Some device behavior is still modeled from observed software behavior
  rather than complete hardware documentation.
